# Trade Analytics - 基本方針

## 目的

マーケット分析・投資戦略作成において、Claude Codeを活用し**分析の手数を増やす**ことを主目的とする。
収益性のある戦略を見つけるには試行錯誤の回数を増やすことが重要であり、そのためにAIエージェントによるリソース圧縮を行う。

> 「Claude Codeを使ってマーケット分析をする」≠「Claude Codeを使ってお金を稼ぐ」
> エージェントはあくまで稼ぐまでの道筋の一部を補助するものである。

## プロジェクト構成

```
├── .claude/
│   └── skills/            # Claude Code Skills定義
├── analyses/              # **marimo notebook** による分析
│   ├── 00_to_be_started/  # 着手前のアイデアストック
│   ├── 01_bookmarks/      # ブックマーク・参考分析
│   └── YYYYMMDD_HHMM_<概要>/  # 個別分析ディレクトリ
│       ├── data/          # 分析固有のデータ
│       ├── img/           # チャート・可視化画像
│       ├── idea_*.md      # 分析アイデアメモ
│       └── analysis_*.py     # marimo分析ノートブック
├── data/                  # マーケットデータ（parquet/duckdb）
├── docs/
│   ├── base.md            # 本ファイル（基本方針）
│   ├── knowledges/        # 蓄積された知見（Markdown）
│   ├── plans/             # 作業計画
│   └── sessions/          # セッション履歴
├── scripts/               # データ取得・分析用スクリプト(pandasではなくpolarsを使用)
├── src/                   # 自作Pythonパッケージ
├── tests/                 # テスト
└── CLAUDE.md              # Claude Code設定
```

## 命名規則

ファイル・ディレクトリは **日時プレフィックス + 分析概要** の形式で命名する。

```
例: 20260301_0930_btc_funding_rate_analysis
例: 20260228_1400_eth_onchain_whale_tracking
```

直近の成果物がディレクトリ末尾に集まり、すぐに見つけられるようにする。

## データ管理

- マーケットデータは **parquet形式** で保存する（軽量・高速・型情報保持）
- 大規模データや複数テーブルの結合が必要な場合は **DuckDB** を使用する
- 分析成果物は **marimo notebook** で管理する

## 知見の蓄積（docs/knowledges/）

以下の情報をMarkdown形式で蓄積し、Claude Codeのコンテキストとして優先的にスキャンさせる。

- 過去の分析から得られた知見・発見
- データ定義・定期取得ジョブの情報
- Pythonパッケージ・スクリプトの使用方法
- 外部API仕様（取引所API、オンチェーンデータAPIなど）

> コンテキストを余計に増やさないよう、ワークスペース全体ではなく knowledges フォルダを優先的にスキャンする。

## 作業計画（docs/plans/）

標準のPlan modeではなく、Skillsを使って `docs/plans/` 配下にMarkdownとして保存する。
IDE内での参照・修正が容易になる。

## 外部AI連携

Claude Code から Skills/MCP を通じて外部AIを呼び出せる。

| ツール    | 用途                           | 備考                          |
| --------- | ------------------------------ | ----------------------------- |
| Codex CLI | 計画立案、コードレビュー       | 計画立案・レビューが得意      |
| Grok API  | リアルタイム市況・ニュース取得 | Xのリアルタイム情報にアクセス |

## 実行環境

- **コンテナ内実行** を前提とする（安全性確保）
- **git によるバージョン管理** を必須とする
- 上記を満たす場合、`--dangerously-skip-permissions` オプションの使用を許容する

## 分析ワークフロー

1. **アイデア出し** - 仮説・着眼点を `idea_*.md` にまとめる
2. **計画立案** - Claude Code（またはCodex CLI）で分析計画を作成し `docs/plans/` に保存
3. **データ準備** - 必要なマーケットデータを取得・加工して `data/` に格納
4. **分析実行** - marimo notebook で分析を実行（Claude Codeが得意）
5. **レビュー** - 結果を検証し、必要に応じて外部AIでレビュー
6. **知見蓄積** - 得られた知見を `docs/knowledges/` に記録
7. **戦略化** - 有望な分析結果を投資戦略・botに落とし込む
