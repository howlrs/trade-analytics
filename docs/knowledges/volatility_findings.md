# ボラティリティ分析の知見

## 堅牢な発見

### 1. ボラティリティ・クラスタリング
- **自己相関**: lag=1h r=0.98, lag=24h r=0.30-0.49 (全トークン)
- ナイーブ予測 (現在vol ≈ 将来vol) のMAE: BTC 0.17%, ETH 0.25%, SOL 0.27%, SUI 0.37%
- MLモデル (LightGBM) はナイーブ予測を上回れなかった

### 2. レジーム遷移行列 (1h step)
```
BTC: L→L=95.6%, M→M=92.0%, H→H=95.8%
```
- 低ボラ・高ボラ共に95%+の持続率
- レジーム変化は稀だが、発生すると数時間〜24h以内に完了

### 3. レジーム別リターン特性 (Test)
| Token | Low Vol | Mid Vol | High Vol |
|-------|---------|---------|----------|
| BTC | -2bp (win 53%) | -8bp (win 50%) | -14bp (win 48%) |
| SOL | +2bp (win 53%) | -30bp (win 45%) | -11bp (win 48%) |

- 低ボラ期は全トークンで勝率が僅かに高い (52-53%)
- ただし8h保有シグナルとしては統計的に非有意

### 4. ボラ圧縮の含意
- ボラ圧縮 → ブレイクアウト仮説は **不成立**
- ボラ圧縮 → 継続低ボラ (スパイク確率が0.3-0.57倍に低下)

### 5. ポジションサイジングの価値
- Vol-adjusted sizing: Sharpe を 0.1-0.2 改善 (微改善)
- Risk Parity vs Equal Weight: -2.12 → -2.08 (微改善)
- **Perfect vol prediction** でも Sharpe は -1.7 → -0.5 程度
- ⇒ **ボラ予測はリスク管理ツールであり、アルファの代替にはならない**

## 実用的な応用

### 裁量トレードのリスク管理
1. **動的ポジションサイズ**: rvol_24h に基づいてサイズを調整
   - position_size = target_risk / (rvol_24h * sqrt(holding_hours/24))
   - 最大レバレッジ上限を設定 (例: 5x)
2. **レジーム意識**: 高ボラ期はサイズ縮小、低ボラ期はサイズ拡大可能
3. **ボラ急変アラート**: rvol_24h が 48h MA の 1.5σ を超えたら警戒

### ポートフォリオ管理
- 4トークンの逆ボラ加重 (Risk Parity) は等ウェイトより微改善
- BTC/ETH → 低ボラ → ウェイト大、SUI → 高ボラ → ウェイト小
