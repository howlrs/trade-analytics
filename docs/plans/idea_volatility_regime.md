# アイデア: ボラティリティ予測 × レジーム分類

## 背景 — なぜ方向予測を諦めるか

3つの独立した手法で方向予測を試み、すべてOOSで崩壊した:
1. **OHLCV線形アルファ** (#9): 五分位分析 → WFで崩壊
2. **デリバティブ主導シグナル** (#11): FR/OI閾値ベース → WFで崩壊
3. **LightGBM非線形探索**: Train Acc 85% → Test Acc 50% (ランダム同等)

共通の根本原因: **1h粒度の8h先リターン方向は、利用可能な特徴量セットでは予測不能**

## 転換: 方向 → ボラティリティ

ボラティリティは方向より予測しやすい (金融工学の定説):
- **ボラティリティ・クラスタリング**: 高ボラは高ボラを呼ぶ (GARCH効果)
- **レジーム持続性**: 低ボラ期間と高ボラ期間は数日〜数週間持続
- **特徴量との関連**: vol_ratio, OI変化, basis, F&Gはレジーム指標として機能する可能性

## 3つの分析ライン

### Line 1: ボラティリティ予測
- ターゲット: 次24h/48hの実現ボラティリティ (ret_1h.rolling_std)
- 連続値回帰 (方向の二値分類ではなく)
- R²が0.3以上なら実用価値あり

### Line 2: レジーム分類
- K-means or HMM で市場レジームを自動検出
- レジーム定義: 低ボラ横ばい / 高ボラトレンド / 高ボラ乱高下
- 各レジームの特性 (持続時間, リターン分布, デリバティブの振る舞い)

### Line 3: レジーム条件付き戦略
- レジームごとに異なる戦略を適用:
  - 低ボラ期: ミーンリバージョン (basis zscore)
  - 高ボラトレンド期: モメンタム (ret_24h + OI increase)
  - 高ボラ乱高下期: ノーポジション
- 過去に崩壊したシグナルも「特定レジームでのみ機能」の可能性

## 必要データ

現在のデータセットで十分:
- OHLCV 1h (8,760行) → ボラ計算
- Bybit OI (1年分) → レジーム特徴量
- FR, Basis → レジーム特徴量
- F&G, Stablecoin → マクロレジーム

追加データ候補:
- Glassnode MVRV Z-score (無料プラン: 日次) → オンチェーンレジーム
- Exchange net flow → 資金動態

## 成功基準

1. ボラティリティR² > 0.3 (OOS)
2. レジーム分類がOOSでも安定 (HMMの状態遷移が一貫)
3. 条件付き戦略でSharpe > 0.5 (Test期間)
