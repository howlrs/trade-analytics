# save-plan

作業計画を `docs/plans/` に保存するスキル。

## トリガー

ユーザーが「計画を保存」「プランを保存」「save plan」と指示した場合、または計画立案が完了した場合に使用。

## 手順

1. 現在の日時を `YYYYMMDD_HHMM` 形式で取得する
2. 計画の概要からディレクトリ名を生成（例: `20260301_0930_btc_analysis_plan`）
3. `docs/plans/<ディレクトリ名>/` を作成
4. 計画内容を連番付きMarkdownファイルとして保存（例: `01_initial_plan.md`）

## 計画ファイルの構成

```markdown
# <計画タイトル>

## 目的
<分析の目的・仮説>

## データソース
<使用するデータ・API>

## 分析手順
1. <ステップ1>
2. <ステップ2>
...

## 期待される成果
<どのような知見が得られるか>

## メモ
<補足事項>
```
